# 栈实现运算求值原理详解

## 一、栈的核心概念

栈是一种**后进先出（LIFO）**的线性数据结构，就像一叠盘子，只能从最顶端放入或取出。

```
栈顶 (Top) ← 唯一操作入口
   │
   ▼
┌─────┐
│  3  │  ← 最后进栈，最先出栈
├─────┤
│  2  │
├─────┤
│  1  │  ← 最先进栈，最后出栈
├─────┤
│  0  │  ← 栈底 (Bottom)
└─────┘
```

### 进栈（Push）操作

```
步骤: 1. 检查栈满 → 2. top++ → 3. stack[top] = element

初始:  top = -1          Push(10):              Push(20):
      [   ]              top = 0                top = 1
      [   ]      →       [10]          →        [20]  ← 栈顶
      [   ]              [  ]                   [10]
      栈底                                       栈底
```

### 出栈（Pop）操作

```
步骤: 1. 检查栈空 → 2. element = stack[top] → 3. top--

状态:  top = 1            Pop():                 Pop():
      [20] ← 栈顶         返回20                 返回10
      [10]        →       top = 0        →       top = -1 (空栈)
      栈底                [10]                   [  ]
                                                   栈底
```

## 二、表达式类型概述

计算机处理表达式有三种形式：

| 类型 | 名称 | 示例 | 特点 |
|------|------|------|------|
| **中缀** | Infix | `3 + 4 * 5` | 人类自然写法，需处理优先级 |
| **前缀** | Prefix（波兰式） | `+ 3 * 4 5` | 运算符在前，从右向左扫描 |
| **后缀** | Postfix（逆波兰式） | `3 4 5 * +` | 运算符在后，从左向右扫描 |

**为什么要转换？**
- 中缀表达式需要处理运算符优先级和括号，实现复杂
- 前缀/后缀表达式无需括号，用栈即可轻松求值

## 三、中缀表达式转后缀表达式

### 3.1 算法原理（Shunting Yard算法）

使用**运算符栈**，从左到右扫描表达式：

| 字符类型 | 处理方式 |
|---------|---------|
| **操作数**（数字/变量） | 直接输出到后缀表达式 |
| **左括号** `(` | 压入栈中 |
| **右括号** `)` | 弹出栈顶运算符并输出，直到遇到左括号 |
| **运算符** `+ - * /` | 与栈顶比较优先级，弹出高优先级运算符后入栈 |

### 3.2 优先级定义

```cpp
int GetPriority(char op) {
  switch (op) {
    case '#': return 0;   // 栈底标记，优先级最低
    case '(': return 1;   // 括号优先级最低
    case '+':
    case '-': return 2;   // 加减
    case '*':
    case '/': return 3;   // 乘除
    case ')': return 4;   // 右括号
    default: return -1;
  }
}
```

### 3.3 转换过程图解

**示例：将 `3 + 4 * 5` 转为后缀表达式**

```
步骤    扫描    操作说明             运算符栈      后缀输出
────────────────────────────────────────────────────────────
初始化  -       压入#作为栈底标记    #            

1       3       输出操作数            #            3

2       +       栈顶# < +, 压栈       # +          3

3       4       输出操作数            # +          3 4

4       *       栈顶+ < *, 压栈       # + *        3 4

5       5       输出操作数            # + *        3 4 5

6       #       弹出栈中所有运算符    #            3 4 5 * +

────────────────────────────────────────────────────────────
结果: 3 4 5 * +
```

**复杂示例：`(3 + 4) * 5 - 6 / 2`**

```
步骤  扫描   操作              栈状态          输出
──────────────────────────────────────────────────
1     (     压栈              # (             
2     3     输出              # (             3
3     +     压栈              # ( +           3
4     4     输出              # ( +           3 4
5     )     弹出到(           #               3 4 +
6     *     压栈              # *             3 4 +
7     5     输出              # *             3 4 + 5
8     -     * > -, 弹出*      # -             3 4 + 5 *
9     6     输出              # -             3 4 + 5 * 6
10    /     - < /, 压栈       # - /           3 4 + 5 * 6
11    2     输出              # - /           3 4 + 5 * 6 2
12    #     弹出所有          #               3 4 + 5 * 6 2 / -
──────────────────────────────────────────────────
结果: 34+5*62/-
```

### 3.4 代码实现

```cpp
void transform(char suffix[], char exp[]) {
  SqStackChar S;                    // 1. 创建运算符栈
  InitStack_Char(S);
  Push_Char(S, '#');                // 2. 栈底标记
  
  char *p = exp;
  char ch = *p;
  int k = 0;
  char c;
  
  while (ch != '#' && ch != '\0') {
    // 跳过空格
    if (ch == ' ' || ch == '\t') {
      ch = *++p;
      continue;
    }
    
    // 情况1: 操作数 → 直接输出
    if (!OpMember(ch)) {
      suffix[k++] = ch;
      ch = *++p;
    }
    // 情况2: 左括号 → 压栈
    else if (ch == '(') {
      Push_Char(S, ch);
      ch = *++p;
    }
    // 情况3: 右括号 → 弹出到左括号
    else if (ch == ')') {
      Pop_Char(S, c);
      while (c != '(') {
        suffix[k++] = c;
        Pop_Char(S, c);
      }
      ch = *++p;
    }
    // 情况4: 运算符 → 比较优先级
    else {
      GetTop_Char(S, c);
      // 栈顶优先级 ≥ 当前，弹出栈顶
      while (c != '#' && precede(c, ch)) {
        suffix[k++] = c;
        Pop_Char(S, c);
        GetTop_Char(S, c);
      }
      Push_Char(S, ch);            // 当前运算符入栈
      ch = *++p;
    }
  }
  
  // 弹出栈中剩余运算符
  while (Pop_Char(S, c) && c != '#') {
    suffix[k++] = c;
  }
  suffix[k] = '\0';
  
  DestroyStack_Char(S);
}
```

## 四、后缀表达式求值

### 4.1 算法原理

使用**操作数栈**，从左到右扫描后缀表达式：

| 字符类型 | 处理方式 |
|---------|---------|
| **操作数** | 压入栈中 |
| **运算符** | 弹出两个操作数，计算后压回栈 |

### 4.2 求值过程图解

**示例：计算 `3 4 5 * +`**

```
步骤    扫描    操作                 操作数栈（栈顶在右）
────────────────────────────────────────────────────────
1       3       压栈                 [3]

2       4       压栈                 [3, 4]

3       5       压栈                 [3, 4, 5]

4       *       弹出5,4              [3]
        ↓       计算4*5=20           
        ↓       压入20               [3, 20]

5       +       弹出20,3             []
        ↓       计算3+20=23          
        ↓       压入23               [23]
────────────────────────────────────────────────────────
结果: 23
```

**复杂示例：`3 4 + 5 * 6 2 / -`**

```
步骤  扫描  操作              栈状态
───────────────────────────────────
1     3     压栈              [3]
2     4     压栈              [3, 4]
3     +     3+4=7             [7]
4     5     压栈              [7, 5]
5     *     7*5=35            [35]
6     6     压栈              [35, 6]
7     2     压栈              [35, 6, 2]
8     /     6/2=3             [35, 3]
9     -     35-3=32           [32]
───────────────────────────────────
结果: 32
```

### 4.3 代码实现

```cpp
double evaluatePostfix(char postfix[]) {
  SqStackDouble S;
  InitStack_Double(S);
  
  for (int i = 0; postfix[i] != '\0'; i++) {
    char ch = postfix[i];
    
    if (isdigit(ch)) {
      // 操作数 → 压栈
      Push_Double(S, ch - '0');
    }
    else if (OpMember(ch)) {
      // 运算符 → 弹出两个数计算
      SElemTypeDouble right, left, result;
      Pop_Double(S, right);    // 先弹出的是右操作数
      Pop_Double(S, left);     // 后弹出的是左操作数
      
      result = Operate(left, ch, right);
      Push_Double(S, result);
    }
  }
  
  SElemTypeDouble finalResult;
  Pop_Double(S, finalResult);
  DestroyStack_Double(S);
  
  return finalResult;
}
```

## 五、前缀表达式求值

### 5.1 算法原理

前缀表达式（波兰式）与后缀类似，但：
- **运算符在操作数之前**
- **扫描方向从右向左**

### 5.2 求值过程图解

**示例：计算 `+ * 2 3 / 8 4`**

含义：`(2 * 3) + (8 / 4) = 6 + 2 = 8`

```
表达式: "+ * 2 3 / 8 4"

从右向左扫描:
┌────────────────────────────────────────────────────────┐
│  步骤  │  字符  │  操作              │  栈状态         │
├────────────────────────────────────────────────────────┤
│   1    │   4    │  压栈              │  [4]            │
│   2    │   8    │  压栈              │  [4, 8]         │
│   3    │   /    │  8/4=2, 压入2      │  [2]            │
│   4    │   3    │  压栈              │  [2, 3]         │
│   5    │   2    │  压栈              │  [2, 3, 2]      │
│   6    │   *    │  3*2=6, 压入6      │  [2, 6]         │
│   7    │   +    │  6+2=8, 压入8      │  [8]            │
└────────────────────────────────────────────────────────┘

结果: 8
```

### 5.3 代码实现

```cpp
double evaluation(char prefix[]) {
  SqStackDouble S;
  InitStack_Double(S);
  
  // 1. 计算表达式长度
  int len = 0;
  while (prefix[len] != '#' && prefix[len] != '\0') {
    len++;
  }
  
  // 2. 从右向左扫描
  for (int i = len - 1; i >= 0; i--) {
    char ch = prefix[i];
    
    if (ch == ' ' || ch == '\t') {
      continue;
    }
    
    if (isdigit(ch)) {
      // 操作数 → 压栈
      Push_Double(S, ch - '0');
    }
    else if (OpMember(ch) && ch != '#' && ch != '(' && ch != ')') {
      // 运算符 → 弹出两个数计算
      SElemTypeDouble a, b;
      if (Pop_Double(S, a) && Pop_Double(S, b)) {
        Push_Double(S, Operate(a, ch, b));
      }
    }
  }
  
  SElemTypeDouble result = 0;
  Pop_Double(S, result);
  DestroyStack_Double(S);
  
  return result;
}
```

## 六、两种求值方式对比

| 特性 | 前缀表达式 | 后缀表达式 |
|------|----------|-----------|
| **运算符位置** | 操作数之前 | 操作数之后 |
| **扫描方向** | 从右向左 | 从左向右 |
| **栈弹出顺序** | 先弹出左操作数 | 先弹出右操作数 |
| **示例** | `+ 3 4` | `3 4 +` |
| **自然度** | 较反直觉 | 较自然 |

## 七、完整运算流程示例

**完整流程：计算中缀表达式 `(3 + 4) * 5 - 6 / 2`**

### 步骤1: 中缀 → 后缀
```
中缀: (3 + 4) * 5 - 6 / 2
      ↓
后缀: 3 4 + 5 * 6 2 / -
```

### 步骤2: 后缀求值
```
扫描: 3 4 + 5 * 6 2 / -

详细过程:
1. 3 入栈                栈: [3]
2. 4 入栈                栈: [3, 4]
3. + 运算: 3+4=7         栈: [7]
4. 5 入栈                栈: [7, 5]
5. * 运算: 7*5=35        栈: [35]
6. 6 入栈                栈: [35, 6]
7. 2 入栈                栈: [35, 6, 2]
8. / 运算: 6/2=3         栈: [35, 3]
9. - 运算: 35-3=32       栈: [32]

结果: 32
```

## 十一、常见问题

**Q1: 为什么要用栈？**
> 栈的LIFO特性完美匹配表达式的嵌套结构，括号匹配和运算符优先级处理都可以优雅地解决。

**Q2: 前缀和后缀哪个更好？**
> 后缀更常用，因为从左向右扫描更符合人类阅读习惯。

**Q3: 如何处理多位数？**
> 需要增加数字解析逻辑，将连续数字字符组合成完整数字后再压栈。

**Q4: 如何处理负数？**
> 可以在数字前加标识符，或在词法分析阶段识别负号。
